Installation

Complete installation guide for Raw2Ready.

System Requirements

  • Python 3.10 or newer
  • Linux, Windows or macOS
  • Minimum 8 GB RAM recommended
  • Internet connection for external knowledge retrieval
  • Optional GPU acceleration for transformer models

Supported Platforms

Operating System Status
Ubuntu Linux Fully Supported
Windows Supported
Windows + WSL Recommended
macOS Supported

1. Clone the Repository


git clone https://gitlab.com/bioindustry4.0/raw2ready.git

cd raw2ready

Alternatively, clone the development branch: 


git clone -b features/mikeantonbranch \
https://gitlab.com/arc3972240/bioindustry/raw2ready.git

cd raw2ready

2. Create a Virtual Environment


python3 -m venv raw2ready

source raw2ready/bin/activate

Windows PowerShell: 


raw2ready\Scripts\activate

3. Install Dependencies


pip install --upgrade pip

pip install -r requirements.txt

Main Dependencies

  • nicegui
  • pandas
  • numpy
  • scipy
  • plotly
  • matplotlib
  • scikit-learn
  • requests
  • beautifulsoup4
  • bacdive
  • crossrefapi
  • spacy
  • nltk
  • ollama

4. Install NLP Resources

Install SpaCy Language Model


python3 -m spacy download en_core_web_sm

Download NLTK Resources


python3


import nltk

nltk.download('punkt')
nltk.download('stopwords')
nltk.download('wordnet')
nltk.download('omw-1.4')
nltk.download('averaged_perceptron_tagger')
nltk.download('vader_lexicon')

5. Install Ollama (Optional)

Raw2Ready supports local execution of Large Language Models using Ollama. 


curl -fsSL https://ollama.com/install.sh | sh

Download a Local Model


ollama pull llama3

Other supported models:

  • llama3
  • mistral
  • gemma
  • phi3

6. Verify Installation

Command Line Interface


python raw2ready.py --help

Launch NiceGUI Application


python run_app.py

Open your browser: 


http://localhost:8080

Optional Docker Deployment


docker build -t raw2ready .

docker run -p 8080:8080 raw2ready

Troubleshooting

Common Problems

  • Python version incompatible.
  • Missing dependencies.
  • SpaCy model not installed.
  • Ollama service not running.
  • Port 8080 already in use.

Check Ollama Status


ollama list

Check Running Processes on Port 8080


lsof -i :8080
