Frequently Asked Questions

Answers to the most common questions regarding installation, configuration, data processing and AI-assisted analysis.

General Questions

Question Answer
What is Raw2Ready? Raw2Ready is a Python-based platform for harmonizing, processing, visualizing and intelligently exploring fermentation datasets using statistical methods, semantic metadata and Large Language Models.
Who is Raw2Ready designed for? Researchers, process engineers, microbiologists, biotechnologists and industrial biotechnology organizations working with fermentation experiments.
Can Raw2Ready be used without programming knowledge? Yes. The NiceGUI graphical interface provides an interactive environment that requires no programming experience for most workflows.
Is Raw2Ready open source? Yes. The source code is available through the project GitLab repository.
Does Raw2Ready require an internet connection? No. Core functionality operates locally. Internet access is only required for external knowledge sources such as BacDive, CrossRef and optional internet search.

Installation

Question Answer
Which Python version is recommended? Python 3.10 or newer.
Can Raw2Ready run on Windows? Yes. Windows, Linux and macOS are supported. Windows Subsystem for Linux (WSL) is recommended for the best compatibility.
Is a GPU required? No. All processing modules work on CPUs. Local LLM inference can optionally benefit from GPU acceleration.
Is Ollama mandatory? No. Ollama is only required when using local Large Language Models.
How do I verify my installation? Run: 
python3 run_app.py
and open the browser at: 
http://localhost:8080

Supported Data

Question Answer
Which file formats are supported? CSV, XLSX, TXT and vendor-specific fermentation exports.
Which equipment is currently supported? Sartorius bioreactors, BioLectorXT systems, gas analyzers and generic spreadsheet data.
Can I import multiple datasets? Yes. Multiple datasets can be loaded and merged using datetime synchronization.
Can Raw2Ready process time-series data? Yes. Time-series processing is one of the core features of the platform.
Can datasets have different sampling rates? Yes. The Merge Module aligns asynchronous measurements using nearest-neighbor matching.

Data Processing

The following section answers common questions regarding preprocessing, merging and calculation modules.

Data Processing Modules

Question Answer
What does the Load Module do? The Load Module imports raw fermentation files, validates their structure and prepares them for preprocessing.
Can I load multiple files simultaneously? Yes. Multiple datasets can be imported during the same session.
What is the Merge Module used for? It synchronizes multiple time-series datasets originating from different instruments.
Which merge strategy is implemented? Nearest-neighbor datetime alignment using a user-selected reference dataset.
What does the Calculate Module do? It evaluates mathematical expressions to generate new variables directly from existing dataset columns.
Can formulas use multiple variables? Yes. Formulas may combine any number of dataset columns together with constants defined in the configuration file.

Visualization

Question Answer
Which plots are available? Line plots, scatter plots, histograms, boxplots, correlation matrices, heatmaps and interactive Plotly visualizations.
Can plots be exported? Yes. Figures can be exported in PNG, SVG, PDF and HTML formats.
Are interactive figures supported? Yes. Interactive Plotly visualizations allow zooming, panning and data inspection.

Statistics

Question Answer
Which statistical analyses are available? Descriptive statistics, correlation analysis, PCA, clustering, ANOVA, regression and hypothesis testing.
Can statistical results be exported? Yes. Statistical tables and figures may be exported for publication.
Does Raw2Ready automatically detect numerical columns? Yes. Numerical variables are identified automatically before analysis.

Metadata & MIFE

Question Answer
What is MIFE? The Minimum Information Framework for Fermentation Experiments standardizes experimental metadata.
Why is metadata important? Metadata provides experimental context that significantly improves AI-assisted interpretation.
Can metadata be exported? Yes. Metadata can be exported in JSON, YAML and MIFE-compatible formats.

AI Agents

Question Answer
Which AI agents are included? Data Analyst, Metadata Analyst, BacDive Explorer, Literature Reviewer, Internet Explorer and Master Summarizer.
Does Raw2Ready use Large Language Models? Yes. Multiple LLMs can be integrated, including local models executed through Ollama.
Can the AI answer questions about my dataset? Yes. AI agents combine experimental data, metadata and biological knowledge to answer natural-language questions.
Is internet access mandatory for AI? No. Local reasoning works without internet access. External services are optional.
What is Runtime Memory? Runtime Memory stores datasets, metadata, statistical results and AI evidence during an active session so agents can share information.

Performance & Runtime

Question Answer
Can Raw2Ready process large datasets? Yes. Raw2Ready is designed to efficiently process large fermentation datasets containing thousands of measurements.
Is all processing performed locally? Yes. Dataset processing, statistical analysis and visualization are executed locally. External services are only used when requested.
Does Raw2Ready save Runtime Memory? Runtime Memory exists only during the current session unless exported by the user.
Can multiple datasets be analyzed in one session? Yes. Multiple datasets may be loaded, merged and analyzed simultaneously.

Export & Reporting

Question Answer
Which export formats are supported? CSV, Excel, JSON, YAML, PDF, PNG, SVG and HTML depending on the module.
Can figures be used in publications? Yes. High-resolution publication-quality figures can be exported.
Can processed datasets be reused? Yes. Harmonized datasets may be exported and reused by other software or machine learning workflows.

Troubleshooting

Problem Recommended Solution
Application does not start Verify that all required Python dependencies have been installed.
Dataset cannot be loaded Check the file format, delimiter and required datetime columns.
Merge operation fails Ensure that all datasets contain valid datetime information.
AI agent returns no response Verify that a dataset has been loaded and Runtime Memory contains data.
BacDive or Literature search fails Verify internet connectivity and the availability of external services.

Best Practices

  • Load and validate datasets before analysis.
  • Complete metadata before using AI agents.
  • Merge datasets using synchronized timestamps.
  • Create calculated variables before visualization.
  • Review statistical results prior to AI interpretation.
  • Use BacDive and Literature modules together for richer biological context.
  • Export processed datasets regularly.
  • Keep Raw2Ready and its Python dependencies updated.
