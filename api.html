Introduction

The Raw2Ready API provides the programmatic interface used throughout the platform for processing, harmonizing, analysing and exploring industrial biotechnology datasets. Although most users interact with Raw2Ready through the NiceGUI graphical interface, every operation is executed by modular Python components that can also be invoked directly from the command line or integrated into custom Python workflows.

The API has been designed around a modular architecture where each processing stage is implemented as an independent component. This facilitates reproducibility, extensibility and straightforward integration of new fermentation equipment, preprocessing algorithms and AI services.

Command Line API

The primary interface for automated processing is the raw2ready.py executable. 


python raw2ready.py [module] [options]

Available Modules

Module Description
parse Parse and harmonize raw fermentation datasets.
merge Synchronize multiple time-series datasets.
calculate Create derived variables using dynamic formulas.

General Syntax 


python raw2ready.py parse      -cf config.json

python raw2ready.py merge      -cf config.json

python raw2ready.py calculate  -cf config.json

Each module uses the same configuration file while only executing the parameters relevant to its own functionality.

Module Execution Order

Although individual modules may be executed independently, the recommended processing pipeline follows the sequence below. 


1. Parse
        ↓

2. Merge
        ↓

3. Calculate
        ↓

4. Visualization
        ↓

5. Statistics
        ↓

6. Metadata Annotation
        ↓

7. AI Exploration

This workflow ensures that every downstream component receives validated, harmonized and semantically enriched experimental data.

Python API

Raw2Ready has been designed using a modular Python architecture. Each processing stage is implemented as an independent module that can be executed individually or combined into larger processing pipelines. 


from modules import parse
from modules import merge
from modules import calculate

These modules act as the entry points for the command line interface and internally delegate processing to the corresponding functions implemented inside the lib package.

Core Library

Library Purpose
lib.parse Equipment-specific parsing and harmonization functions.
lib.merge Datetime synchronization and nearest-neighbor merging.
lib.calculate Formula execution engine.
lib.utilities Common helper functions shared by every module.

Configuration Files

Every processing module is driven by external configuration files. This allows experiments to be reproduced without modifying the source code.

File Description
config.json Main experiment configuration.
calculate.config.yaml Formula definitions and constants.
column_translation.json Maps raw sensor names to standardized column names.
MIFE metadata Experimental metadata and cultivation information.

Extension Points

Raw2Ready has been designed as an extensible platform. New functionality can be incorporated without modifying the overall processing workflow.

Component Extension Strategy
New Equipment Add new parsing functions in lib/parse.py and register them in modules/parse.py.
New AI Agent Create an additional agent inside the agents/ package and connect it to Runtime Memory.
New Visualization Extend the plotting module with additional Plotly or Matplotlib figures.
New Statistical Method Integrate the corresponding analysis inside the Statistics module.
New Metadata Schema Extend the MIFE metadata model while maintaining backward compatibility.

Best Practices

  • Keep parsing logic independent of the graphical interface.
  • Reuse the existing library functions whenever possible.
  • Always validate datasets before downstream analysis.
  • Preserve standardized column naming conventions.
  • Update configuration templates when introducing new functionality.
  • Maintain compatibility with Runtime Memory.
  • Document every new module before release.
  • Keep the GitLab repository synchronized with the documentation.
