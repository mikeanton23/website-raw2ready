Introduction

Raw2Ready has been designed to integrate heterogeneous datasets generated throughout industrial biotechnology workflows. Rather than requiring users to manually convert files, the platform provides dedicated parsers for supported laboratory equipment and generic tabular formats.

All imported datasets are transformed into a standardized internal representation before subsequent processing modules—including merging, calculations, visualization, statistical analysis, metadata annotation and AI-assisted exploration—are executed.

The goal of the supported format architecture is to provide a consistent and reproducible workflow regardless of the original vendor-specific data format.

Standardized Dataset

Regardless of the original source, every imported dataset follows the same harmonization pipeline before becoming available to the remaining Raw2Ready modules.

Why Standardized Formats Matter

Industrial biotechnology experiments frequently combine measurements from multiple instruments that use different file formats, timestamp conventions and variable naming schemes.

Raw2Ready removes these inconsistencies by converting all supported datasets into a unified internal representation, allowing downstream modules to operate independently of the original equipment.

Validation Rules

Every imported dataset is validated before processing.

  • File exists.
  • Supported extension.
  • Readable encoding.
  • Valid delimiter.
  • Required columns present.
  • Consistent datetime format.
  • Duplicate timestamps detected.
  • Missing values identified.
  • Column types verified.
  • Output directory accessible.

Missing Values

Missing observations frequently occur in industrial fermentation datasets due to asynchronous measurements, sensor failures or acquisition interruptions.

Raw2Ready identifies missing values during import and reports them before downstream processing begins.

Condition Action
Empty cells Detected and reported.
NaN values Preserved unless explicitly handled.
Duplicate timestamps Warning generated.
Invalid numeric values Validation warning.

Knowledge Discovery

Every exported dataset preserves standardized naming, timestamps and metadata, ensuring compatibility throughout the complete Raw2Ready workflow.

Best Practices

  • Keep original raw files unchanged.
  • Store configuration files together with each experiment.
  • Preserve standardized column names.
  • Always validate datasets before merging.
  • Maintain consistent timestamp formats.
  • Include complete MIFE metadata whenever possible.
  • Use UTF-8 encoding for text-based files.
  • Archive generated output files for reproducibility.

Common Formatting Problems

Problem Recommended Solution
Different datetime formats Convert timestamps before merging.
Duplicate measurements Remove duplicates or inspect manually.
Missing variables Verify parser and column translation.
Incorrect encoding Save files using UTF-8.
Inconsistent units Standardize measurement units.

Troubleshooting

If an import fails, verify the following:

  1. The selected equipment parser matches the source instrument.
  2. The file extension is supported.
  3. The configuration file references the correct paths.
  4. Required columns are present.
  5. Datetime values are valid.
  6. Files use UTF-8 encoding whenever possible.
  7. Output directories are writable.

Detailed diagnostic information is available in the corresponding module log files.
