Introduction

This section demonstrates complete experimental workflows using Raw2Ready. The examples illustrate how heterogeneous raw files are transformed into standardized datasets, visualized, statistically analysed and explored through AI-assisted reasoning.

Each example follows the same philosophy used throughout the documentation and can be adapted to different experimental configurations.

Step 1 — Parse Raw Data 


python raw2ready.py parse \
    -cf config.json

The parser imports raw Sartorius sensor exports and gas analyser measurements, applies column translations, converts timestamps into the standardized internal format and generates harmonized CSV datasets for downstream processing.

Expected Output

Generated File Description
experiment.sensors.csv Parsed sensor measurements.
experiment.gas_analyzer.csv Parsed gas analyser dataset.
parse.log Execution log.

Once harmonized, the dataset is ready for merging with additional experimental measurements.

Complete Command-Line Workflow

A complete experiment can be processed using three consecutive commands. 


python raw2ready.py parse \
    -cf config.json

python raw2ready.py merge \
    -cf config.json

python raw2ready.py calculate \
    -cf config.json

Once calculations are completed, the resulting dataset can be explored through the Plot, Statistics, Metadata and AI modules.

Generated Files

File Description
*.sensors.csv Parsed sensor dataset.
*.gas_analyzer.csv Parsed gas analyser dataset.
*.merged.csv Time-synchronized dataset.
*.calculate.csv Dataset containing calculated variables.
*.log Processing logs.

Best Practices

  • Keep the original raw files unchanged.
  • Use descriptive experiment names.
  • Validate configuration files before processing.
  • Review parser log files after each execution.
  • Maintain complete metadata for every experiment.
  • Verify calculated variables before publication.
  • Archive configuration files together with datasets.
  • Store generated plots and statistics alongside processed data.

Troubleshooting

Issue Recommendation
Parser failed Verify the configuration file and input paths.
Merge failed Inspect timestamp formats and reference dataset.
Formula errors Check aliases and YAML expressions.
Missing variables Review the column translation configuration.
Unexpected plots Confirm that numerical variables were imported correctly.
