Configuration Reference

JSON Configuration • YAML Formulas • Metadata • Runtime Parameters

Introduction

Every processing module inside Raw2Ready is driven by external configuration files. Instead of modifying Python source code, users specify experiment information, input files, processing parameters and output settings through structured JSON and YAML documents.

This design separates configuration from implementation, making workflows reproducible, portable and easy to share between researchers and laboratories.

The same configuration philosophy is used by every Raw2Ready module, including parsing, merging, calculations, metadata annotation and AI-assisted analysis.

Main Configuration File

The primary configuration file is a JSON document that contains both global experiment parameters and module-specific configuration blocks. 


config.json

├── output directory
├── experiment name
├── experiment start date
├── equipment
│
├── parse parameters
│
├── merge parameters
│
└── calculate parameters

Each module accesses only the corresponding section of the configuration file while ignoring unrelated parameters.

Example Configuration Structure 

{
    "output directory": "...",

    "experiment name": "...",

    "experiment start date": "...",

    "equipment": "...",

    "parse parameters": {

    },

    "merge parameters": {

    },

    "calculate parameters": {

    }
}

Although this represents the minimum structure, each module can contain additional optional parameters depending on the supported equipment and processing workflow.

Global Parameters

Parameter Description
output directory Destination directory where all generated files, logs and reports will be stored.
experiment name Unique identifier used for naming output files and log files.
experiment start date Reference date used by equipment requiring absolute timestamps.
equipment Specifies which equipment parser should be executed.

Parse Parameters

The parse parameters block defines all information required for importing and harmonizing raw experimental data. The exact parameters depend on the selected equipment. 


"parse parameters": {

    "sensor input files": "...",

    "sensor column translation file": "...",

    "sensor output file": "...",

    "gas analyzer input file": "...",

    "gas analyzer target channel": 3,

    "gas analyzer reference channel": 6,

    "gas analyzer output file": "..."

}

Parse Parameter Reference

Parameter Description
sensor input files One or more raw sensor files. Multiple files are separated by commas.
sensor column translation file JSON file mapping vendor-specific names to standardized column names.
sensor output file Output CSV generated after parsing.
gas analyzer input file Raw gas analyzer export.
gas analyzer target channel Channel corresponding to the fermentation experiment.
gas analyzer reference channel Reference gas channel.
gas analyzer output file Parsed gas analyzer dataset.

Merge Parameters

The Merge Module combines multiple time-series datasets using datetime synchronization. 


"merge parameters": {

    "reference input file": "...",

    "input files": "...",

    "output file": "..."

}

Merge Parameter Reference

Parameter Description
reference input file Dataset whose datetime column is used as the synchronization reference.
input files One or more datasets that will be aligned to the reference dataset.
output file Final merged dataset.

Calculate Parameters

The Calculate Module evaluates mathematical expressions defined in a YAML configuration file and creates new dataset columns. 


"calculate parameters": {

    "calculate config file": "...",

    "input file": "...",

    "output file": "..."

}

Calculate Parameter Reference

Parameter Description
calculate config file YAML file containing constants, aliases and mathematical formulas.
input file Dataset used for formula execution.
output file Extended dataset containing newly generated variables.

Column Translation File

Raw equipment exports often use vendor-specific variable names. The column translation file maps those names to standardized, human-readable variables. 

{
    "PDatTime":"PDatTime",

    "Age (h)":"Age (h)",

    "pH:Value":"pH [/]",

    "TEMP:Value":"Temperature [°C]",

    "PO2:Value":"pO2 [%]",

    "STIR:Value":"Agitation [rpm]"
}

The left-hand side contains the original equipment variables, whereas the right-hand side specifies the standardized names that will appear throughout the platform.

YAML Formula Configuration

Dynamic calculations are defined using a dedicated YAML configuration file. 


constants:

    den_ph_ctrl_sol: 0.9425

    starting_feed: 0.5


aliases:

    base_weight: "Base Weight [g]"

    oxygen: "Oxygen [%]"

    substrate: "Substrate Feed [%]"


formulas:

- name: rel_feed

  alias: "Relative Feed"

  formula: "substrate-starting_feed"

  required_columns:

      - "Substrate Feed [%]"

YAML Blocks

Block Description
constants Experiment-specific numerical values.
aliases Maps column names to Python variables.
formulas Mathematical expressions executed during processing.

MIFE Metadata Configuration

Raw2Ready supports semantic annotation through the Minimum Information Framework for Fermentation Experiments (MIFE). Metadata accompanies every processed dataset and enables reproducibility, interoperability and AI-assisted reasoning. 

{
    "experiment": {

        "title": "Fed-Batch Fermentation",

        "organism": "Escherichia coli",

        "strain": "BL21(DE3)",

        "medium": "Defined Medium",

        "bioreactor": "Sartorius",

        "operator": "Researcher",

        "start_date": "2025-06-04"

    }
}

Metadata are stored independently from numerical measurements and become available to the AI framework, statistical modules and literature retrieval agents.

Best Practices

  • Keep configuration files under version control.
  • Avoid modifying generated output files manually.
  • Use descriptive experiment names.
  • Keep metadata complete and consistent.
  • Validate formulas before executing calculations.
  • Maintain standardized column names across all datasets.
  • Store configuration files together with raw data whenever possible.
  • Update configuration templates when new equipment is introduced.

Common Configuration Errors

Error Possible Cause
Missing input file Incorrect file path.
Unknown equipment Unsupported parser selected.
Invalid formula Syntax or missing variables.
Missing column Column translation mismatch.
Datetime mismatch Incorrect timestamp formatting.
Metadata validation failed Missing required MIFE fields.

Troubleshooting

If configuration problems occur, verify the following:

  1. The configuration file is valid JSON.
  2. All referenced files exist.
  3. Equipment names match supported parsers.
  4. Output directories are writable.
  5. YAML formulas contain valid expressions.
  6. Required metadata fields are present.

Detailed diagnostic information is available in the application log files generated during execution.
